> **报告说明**：本报告为有 PPO/DQN 基础的求职者，针对(1) 国内游戏AI岗（腾讯、网易伏羲、米哈游、莉莉丝、超参数等），(2) 通用 RL 算法工程师岗，(3) AI Agent 开发岗 三个方向，提供基于2025–2026年最新行业JD与论文的项目方案。报告分为四部分：市场需求精准画像、可立即落地的项目方案、差异化加分要点、学习路径设计。所有结论尽量给出公开来源链接。报告中涉及"建议/可能"的内容均为分析判断，非定论。

### 1.2 三类岗位的"考点交集"——HR/技术面试官最看重什么

按腾讯AI Lab前研究员、知乎众多面经及实习经验贴汇总，深度RL面试覆盖的核心考点如下 ([Source](https://zhuanlan.zhihu.com/p/186093914))：

| 考察维度 | 高频问题 |
|---|---|

| **算法深度** | dual-clip PPO、IMPALA(V-trace)、distributional RL（C51/QR-DQN）、内在奖励/RND、模仿学习(BC/GAIL/AIRL) |
| **多智能体** | CTDE、QMIX、MAPPO、COMA、credit assignment、MARL non-stationarity |
| **博弈论** | NE、CFR、CFR+、Deep CFR、NFSP、self-play、Population-Based Training |
| **工程系统** | 分布式RL架构（Actor-Learner、Reanalyze）、Ray/RLlib、采样吞吐与GPU利用率、奖励工程与课程学习 |
| **LLM时代新增** | RLHF / DPO / GRPO 区别、ReAct / Reflexion / Tree-of-Thought、长Horizon Agent训练、tool use、多Agent协作 |

王者荣耀绝悟实习面试者反馈，面试问到了 DDPG → TD3 → SAC 的演进、网络结构（特征工程、channel attention、self-attention、multi-head value estimation）、奖励函数设计、**dual-clip PPO + NoisyNet**、自对弈模型筛选、策略集成等系列问题 ([Source](https://www.nowcoder.com/creation/subject/a71a422c110446c88965ffa4867db761))。

字节AI Lab/腾讯AI Lab 实习面试经验明确总结了面试官最看重三类东西——**(a) 顶会论文或细分领域论文熟悉度，(b) 实际落地项目（项目越多工程实践越强），(c) 对常见算法（PPO、IMPALA）的"细节级"掌握**，并且面试官还会出场景题，例如"如果让你做NBA 2K21的游戏AI，你的技术路线是什么、人员分工怎样" ([Source](https://zhuanlan.zhihu.com/p/186093914))。简历方面，"做过什么、为什么这么做、改进点是什么、消融了什么"是必谈四问；说"项目细节保密"会减分 ([Source](https://zhuanlan.zhihu.com/p/44746720))。

### 1.3 关键趋势判断（用于项目主题选择）

1. **游戏AI已进入"RL + LLM 混合架构"时代。** 学界2025年的综述明确指出未来必然是Hybrid：宏观层 MCTS/Director Systems，战术层黑板系统，微观层行为树/HTN，**交互层嵌入 LLM 提供叙事深度，底层用 DRL 训练特定操作** ([Source](https://zhuanlan.zhihu.com/p/1978197400863653976))。Tencent 2025 论文 PORTAL 直接用 LLM 生成行为树DSL，结合 RL 神经网络节点，号称能玩"几千个 3D 游戏" ([Source](https://arxiv.org/html/2503.13356v1))。LLM 驱动 NPC 已在《传奇世界》《幻塔》等商业游戏落地（如盛趣"玄玄老人"接入 DeepSeek） ([Source](https://m.mp.oeeee.com/a/BAAFRD0000202503261063094.html))。
2. **LLM Agent 训练正走向"Agentic RL"。** 2025年发布的 AgentGym-RL（复旦） ([Source](https://agentgym-rl.github.io/))、Agent-R1 ([Source](https://arxiv.org/html/2511.14460v1))、SPIRAL（NUS/Sea） ([Source](https://arxiv.org/abs/2506.24119)) 都把 LLM Agent 作为多回合RL的policy训练，PettingZoo/RLCard等"传统RL环境"反而成为训练LLM推理的环境基础。
3. **RLHF → RLVR/GRPO/DAPO 是post-training核心方向。** DeepSeek-R1 引爆的 GRPO 已成 LLM 领域当年最大算法热点之一 ([Source](https://magazine.sebastianraschka.com/p/state-of-llms-2025))，DAPO（字节/清华2025）在 AIME 2024 上让 Qwen2.5-32B 用50%更少步数超过 R1-Zero ([Source](https://llm-stats.com/blog/research/post-training-techniques-2026))。
4. **传统"MOBA/MARL"项目天花板很高。** 王者绝悟5v5使用上千CPU核+多GPU+ Ray级别分布式 ([Source](https://zhuanlan.zhihu.com/p/541955662))，普通校招求职者无法复现完整规模，因此**"完整复现"不是亮点，"在小规模上引入新机制+严谨消融"才是亮点**。

### 2.2 推荐的"主项目"——合三为一的旗舰项目

> **项目代号：HybridArena —— 一个开源的"LLM-决策 × DRL-控制"混合智能体训练与基准平台**
> 
> **一句话定位：** 在一个不完全信息策略小游戏（如简化版斗地主或自定义 MOBA-mini）中，用 LLM 做高层宏观策略与对话/谈判，用 PPO/MAPPO/MuZero 做低层动作控制，并完整复现 SPIRAL 风格 self-play 训练 LLM 推理能力 + 王者绝悟风格 dual-clip PPO 训练微观操作的混合管线。

**为什么这一项目近乎完美：**

1. **同时命中三个赛道**：游戏AI（混合架构是2025–2026公认趋势 ([Source](https://zhuanlan.zhihu.com/p/1978197400863653976))）、通用RL（涉及PPO/MAPPO/CFR/Self-Play 等几乎所有面试核心算法）、LLM Agent（直接用ReAct/Reflexion 风格 LLM 作 high-level planner并做 Multi-turn RL fine-tune）。
2. **项目独特性**：截至 2025 年底，"LLM做高层规划 + RL做低层控制"在学术上有 PORTAL ([Source](https://arxiv.org/html/2503.13356v1))、LGC-MARL ([Source](https://www.classicinformatics.com/blog/how-llms-and-multi-agent-systems-work-together-2025))、SPIRAL ([Source](https://arxiv.org/abs/2506.24119)) 等论文，但在校招圈仍然非常稀缺，能立刻和"千篇一律的 Atari/CartPole 项目"区分开。
3. **可控的复现规模**：不需要1000核CPU也能产出有意义的Ablation，因为环境本身可设计成中等规模（PettingZoo 接口或自研轻量 MOBA-mini）。

**模块拆分（建议四阶段，每阶段都能独立写进简历）：**

| 阶段 | 内容 | 写进简历的"亮点"（建议表述） |
|---|---|---|
| **A. 自定义环境与 Baseline** | 用 PettingZoo Parallel API 搭一个 4v4 不完全信息小型 MOBA（或简化斗地主对战）；提供 Gymnasium 单智能体 wrapper | "基于 PettingZoo 1.24 自研 4v4 不完全信息环境，支持 PPO/MAPPO/QMIX/CFR/MuZero 五大算法统一接入，每秒 6k+ env steps（8 envs）" |
| **B. 微观控制 RL 强化** | 在 CleanRL 基础上实现 **dual-clip PPO**（绝悟）+ self-play + curriculum learning + reward shaping | "复现腾讯AAAI 2020 dual-clip PPO并集成 Hugging Face DRL 课程的自对弈机制 ([Source](https://huggingface.co/learn/deep-rl-course/en/unit7/self-play))；与 vanilla PPO 相比 ELO 提升 +XXX" |
| **C. 高层 LLM 规划与对话** | 用 LangGraph 编排一个 ReAct/Reflexion 风格 high-level planner（决策"何时开团/分推"），调用底层 RL policy 作工具；对话部分接 Qwen-7B 或 DeepSeek 做风格化 NPC | "采用 LangGraph 状态机做高层规划，CrewAI 做多 NPC 角色协作 ([Source](https://www.datacamp.com/tutorial/crewai-vs-langgraph-vs-autogen))；解决了纯LLM agent 在策略游戏中常见的'活在文本里'问题" |
| **D. Agentic RL 训练（核心创新）** | 基于 SPIRAL/AgentGym-RL 框架对 LLM planner 做 multi-turn GRPO 微调，并和 Role-Conditioned Advantage Estimation 一起训练 | "复现 SPIRAL 的 RAE 并迁移到 4v4 场景，GRPO 训练 Qwen3-4B planner，整体胜率比纯 prompt-based ReAct 提升 8.X% ([Source](https://arxiv.org/html/2506.24119v1))" |

**完整对照实验矩阵（这是面试官最爱看的）：**

- 微观层：PPO / dual-clip PPO / RPO / MAPPO / QMIX / MuZero(简) 在同一环境的胜率与训练曲线
- 高层：rule-based / vanilla LLM ReAct / LLM+RAG / LLM+GRPO 的胜率对比
- 对照基准：纯 RL（无 LLM）的 ELO，纯 LLM（无 RL）的 ELO，Hybrid 的 ELO
- 超参敏感性：clip ε、KL 系数、self-play 历史池长度
- 可视化：TensorBoard + W&B + 用 streamlit 做一个"AI 解说员"实时显示 LLM 的 chain-of-thought 与 RL agent 的Q值

### 2.3 "副项目"建议保留 1–2 个突出深度

为了简历兼顾"广度+深度"，建议主项目之外额外保留 **2** 个深度专项小项目：

**副项目 1：Deep CFR + ESCHER on Leduc/FHP（对应博弈论深度）**

- 完全自实现 Deep CFR ([Source](https://arxiv.org/pdf/1811.00164))，再叠加 ESCHER 的零方差估计器 ([Source](https://www.mit.edu/~gfarina/2023/escher_iclr23/2206.04122.pdf))
- 用 OpenSpiel 提供的 Kuhn/Leduc 作 baseline，FHP（Flop Hold'em Poker）做主战场
- 写入简历："比 OpenSpiel 内置 Deep CFR exploitability 降低 X%，引入GPU向量化使 1k iter 提速 22× ([Source](https://arxiv.org/pdf/2408.14778))"
- 这个项目对腾讯棋牌、网易棋牌、超参数科技岗位有**直接业务匹配**

**副项目 2：基于 LightZero 的 Stochastic MuZero 在不完全信息小游戏上的尝试（对应Model-based RL）**

- LightZero 是 NeurIPS 2023 Spotlight 中文团队（OpenDILab）的MCTS+RL 统一基准 ([Source](https://github.com/opendilab/LightZero))
- 复现 EfficientZero V2 的 Sampled Gumbel Search ([Source](https://arxiv.org/html/2403.00564v2))，并在自定义环境上做 ablation
- 写入简历："实现 EfficientZero V2 的 self-supervised consistency loss 与 value-prefix LSTM，在 Atari-100k 子集上对比 vanilla MuZero 样本效率提升 X×"
- 这个项目能秀出"懂 model-based RL & MCTS"，是当年DeepMind/伏羲研究员都看重的硬通货

> **简历项目结构建议：** 主项目 HybridArena（占简历最大篇幅，体现广度+趋势）+ Deep CFR/ESCHER（体现博弈论深度）+ LightZero/MuZero V2（体现 model-based + 工程深度）。三个项目合起来覆盖 PPO、MAPPO、QMIX、MuZero、Deep CFR、Self-play、GRPO、LLM Agent 整套面试考点。

### 2.4 关键技术栈研究小结（用于撰写"我为什么这么做"）

**(1) ML-Agents PPO 改进方向：** ML-Agents 官方训练器现已支持 PPO/SAC/POCA 三种 ([Source](https://unity-technologies.github.io/ml-agents/Training-Configuration-File/))，2025 年综述指出 PPO 在游戏复杂场景胜率 94.7%，远高于行为树 82.3% 与 FSM 76.9% ([Source](https://johal.in/game-development-ai-implementation-with-unity-ml-agents-and-python-api-2026/))。**改进方向**：(a) shared critic for PPO（ML-Agents 已支持 ([Source](https://github.com/Unity-Technologies/ml-agents/releases?after=0.8.1))），(b) curriculum learning（已有研究表明可减 45% 训练时间提升 32% 性能 ([Source](https://johal.in/game-development-ai-implementation-with-unity-ml-agents-and-python-api-2026/))），(c) 模型量化用 ONNX 推理时延从 150ms 降到 8ms，是工业级关键卖点。

**(2) PettingZoo MARL credit assignment：** AAMAS 2025 的 PyMARLzoo+ 基准把 QMIX、QPLEX、HAPPO、MAT-DEC、CDS、EOI、EMC、MASER 全部整合进同一 PettingZoo 接口，结论是"QMIX 在 PettingZoo 任务上仍是最强；QPLEX 在大多任务上稳超 QMIX；MAA2C/MAPPO 在稀疏奖励 LBF 上最稳" ([Source](https://www.ifaamas.org/Proceedings/aamas2025/pdfs/p1613.pdf))。COMA 的 counterfactual baseline、QMIX 的 monotonic mixing 仍是 credit assignment 两大经典方案 ([Source](https://arxiv.org/pdf/2111.01100))。2023 年 PC-MAPPO 提出基于"预测贡献度量"的 credit assignment，在 SMAC 多个 super-hard 地图超过 MAPPO/QMIX/WQMIX ([Source](https://www.sciencedirect.com/science/article/abs/pii/S0893608023002654))。

**(3) MuZero 简化复现路线：** 最被认可的开源教学版本是 muzero-general（PyTorch + Ray，CartPole/Connect4/Atari 都跑通） ([Source](https://github.com/werner-duvaud/muzero-general))。最值得加的两个改进是 **EfficientZero**：(a) 自监督一致性损失防止 latent 表征塌缩，(b) value-prefix LSTM 缓解奖励对齐误差，能在 Atari 100k 上达到 190.4% mean human performance ([Source](https://arxiv.org/html/2403.00564v2))。如果要做棋牌，**Stochastic MuZero**（决策节点+chance节点分离）才能正确建模隐藏信息 ([Source](https://openreview.net/forum?id=X6D9bAHhBQ1))。

**(4) CleanRL 改进：** CleanRL 单文件 PPO 仅340 行 ([Source](https://github.com/vwxyzjn/cleanrl))，最显著的改进可以是：(a) 切到 EnvPool 实现 3–4× 加速 ([Source](https://docs.cleanrl.dev/rl-algorithms/ppo/))，(b) 多GPU PPO（ppo_atari_multigpu.py 比单GPU SyncVectorEnv 快 34%），(c) 切换到 RPO（Robust Policy Optimization）在 93% 环境上不差于或好于PPO ([Source](https://docs.cleanrl.dev/rl-algorithms/rpo/))，(d) 实现 invalid action masking。CleanRL 哲学就是"每个改进都能 file-diff 看到"，对面试官展示能力极佳 ([Source](https://www.jmlr.org/papers/volume23/21-1342/21-1342.pdf))。

**(5) 不完全信息博弈（斗地主/德州/麻将）：**
- DouZero（快手 ICML 2021）：用最朴素的 Deep Monte-Carlo + LSTM 编码历史动作 + 27,472 维动作空间编码成 4×15 牌矩阵，4-GPU服务器训练数天即超越所有现存斗地主AI，是Botzone第一名 ([Source](https://arxiv.org/html/2106.06135))。其代码已集成到RLCard ([Source](https://github.com/datamllab/rlcard))。
- CFR 系列：CFR+ 解决了 Limit Texas Hold'em ([Source](https://arxiv.org/pdf/1901.07621))；Deep CFR 用神经网络替代 abstraction 解决大型游戏 ([Source](https://arxiv.org/pdf/1811.00164))；ESCHER 通过去除 importance sampling 显著降低估计器方差 ([Source](https://www.mit.edu/~gfarina/2023/escher_iclr23/2206.04122.pdf))；GPU加速 CFR 在大型游戏上比 OpenSpiel C++ 快 22.2× ([Source](https://arxiv.org/pdf/2408.14778))。

**(6) LLM-based Agent 框架对比：** 截至 2026 年初，LangGraph（图状态机，27,100月搜索量）、CrewAI（角色制，14,800月搜索量）、AutoGen/AG2（会话制）、OpenAI Agents SDK（handoff 制）、Google ADK（层级树）、Anthropic Claude Agent SDK（工具链+sub-agent）形成六足鼎立 ([Source](https://gurusup.com/blog/best-multi-agent-frameworks-2026))。最契合"游戏多NPC协作"的组合是 **LangGraph（高层状态机）+ CrewAI（NPC 角色化）**；纯研究/Agentic RL 训练则首选 AgentGym-RL ([Source](https://agentgym-rl.github.io/)) 或 Agent-R1 ([Source](https://arxiv.org/html/2511.14460v1))，后者是 GitHub 上首个真正端到端开源 multi-turn RL LLM Agent 训练平台。

**(7) 游戏中"LLM对话+RL战斗"混合架构案例：**
- **PORTAL（Tencent 2025）**：LLM 生成行为树DSL，节点可以是神经网络也可以是规则，号称玩"千个3D游戏" ([Source](https://arxiv.org/html/2503.13356v1))。
- **Voyager（NVIDIA Caltech 2023, ICLR Outstanding）**：GPT-4驱动 Minecraft agent，自动课程+技能库+迭代提示，比SOTA多3.3×新物品、解锁钻石速度快 15.3× ([Source](https://voyager.minedojo.org/))。
- **PANGeA、Inworld AI**：成熟商业NPC对话生成解决方案，强调情境感知 ([Source](https://zhuanlan.zhihu.com/p/1978197400863653976))。
- **SPIRAL（NUS/Sea AI Lab 2025）**：让 LLM 通过零和小游戏自对弈学到推理能力，仅用 Kuhn Poker 训练即让 Qwen3-4B 数学推理提升 8.6% ([Source](https://arxiv.org/html/2506.24119v1))，这个思路反过来用就是"用 LLM 让游戏 AI 更会规划"。

## 第三部分：如何让项目"脱颖而出" —— 区分千篇一律的 Atari/CartPole 项目

### 3.1 让面试官眼前一亮的"5 张牌"

总结 Tencent / 商汤 等在职面试官在知乎/牛客的明确反馈，一个"加分"项目应同时具备 ([Source](https://zhuanlan.zhihu.com/p/44746720)) ([Source](https://zhuanlan.zhihu.com/p/186093914))：

1. **业务相关性（Business Relevance）。** Atari / CartPole 已是2018年门槛，2026 年门槛是"贴近真实业务"。游戏AI岗最相关的业务是：MOBA 1v1、4v4 副本、风格化 NPC、棋牌不完全信息、自动化测试、难度自适应、AI 反外挂。
2. **技术深度（Technical Sophistication）。** 必须出现一个具体改进点而不是"调包跑通"。例如绝悟之于PPO的改进是 dual-clip PPO，DouZero 的改进是 DMC + LSTM 编码动作矩阵，**你的项目必须有一个有名字的改进**。
3. **端到端实现（End-to-end）。** 不止训练，还要包括环境、训练框架、推理部署、可视化、A/B 评测。绝悟 5v5 体系强调"高可扩展低耦合的强化训练系统"作为核心贡献之一 ([Source](https://blog.csdn.net/weixin_51426083/article/details/131056928))。
4. **严谨消融（Rigorous Ablation）。** 王者绝悟论文专门对比了 Base+AM、Base+TA、Base+LSTM、Full version 四组 ([Source](https://zhuanlan.zhihu.com/p/342391256))；这种 ablation table 写到简历项目说明里非常加分。
5. **"自己的思考"（Personal Insight）。** 从面试官视角："为什么这么做？换X方法会怎样？是否尝试过？"是必问三连 ([Source](https://zhuanlan.zhihu.com/p/27537084854))；一个加分项目必须每个设计选择都有一段笔记/博客解释。

## 第四部分：从 PPO/DQN 基础到能完成项目的完整学习路径

> **总思路**：4–6 个月，分基础-进阶-前沿三阶段，每阶段都对应一个可写简历的成果。

### 4.1 阶段一（4–6 周）：把 PPO/DQN "吃透"到能在面试中默写
